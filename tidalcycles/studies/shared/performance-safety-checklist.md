# Performance Safety Checklist

Use this before teaching or performing any artist-code study.

## Before Starting

- Confirm audio output and SuperDirt are running.
- Run the first study file's sound check before opening denser examples.
- Keep a stable `d1` anchor ready.
- Keep `hush` visible and easy to evaluate.
- Start with conservative gain values.

## During Dense Sections

- Change one dimension at a time: rhythm, sample, pitch, speed, effect, or density.
- Avoid adding delay feedback and density increases at the same time.
- Keep sliced, stuttered, and delayed layers quieter than anchors.
- If the groove is lost, silence the most transformed layer first.
- Return to a known-good anchor before introducing a new idea.

## Ending Practice

- Rehearse one subtractive ending.
- Rehearse one global color change.
- Rehearse `hush` as the final emergency stop.
- Practice ending within four cycles.

## Recovery Commands

```tidal
d4 $ silence
d3 $ silence
d2 $ silence
d1 $ sound "bd ~ ~ ~" # gain 0.65
hush
```
