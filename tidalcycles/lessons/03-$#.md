# `$` and `#`

Tidal lines often use two operators that do very different jobs:

- `$` applies a complete expression to a function.
- `#` combines control patterns.

## `$`

`$` comes from Haskell. It means function application with very low
precedence:

```haskell
f $ x
```

is the same as:

```haskell
f x
```

In Tidal, it mostly saves parentheses. ([Tidal Cycles][4], [Haskell][5])

```haskell
d1 $ s "bd sn"
```

means:

```haskell
d1 (s "bd sn")
```

With transformations:

```haskell
d1 $ every 3 rev $ s "bd sn"
```

means:

```haskell
d1 (every 3 rev (s "bd sn"))
```

Read `$` as: finish the expression on the right, then pass it into the
function on the left.

## `#`

`#` is a Tidal pattern-combination operator. It is an alias for `|>`.
It keeps the event structure from the left pattern and maps the right-side
controls onto that structure. ([Tidal Cycles][1], [Tidal Cycles][2])

```haskell
d1 $ s "bd sn" # gain "1 0.5 0.8"
```

The sound pattern supplies the rhythm. The gain pattern supplies gain values
for those events.

If the order changes, the structure can change:

```haskell
d1 $ gain "1 0.5 0.8" # s "bd sn"
```

Now the gain pattern sits on the left, so its structure is dominant.

## Controls Are Patterns

`sound`, `gain`, `pan`, `begin`, `end`, `delay`, and similar functions create
control patterns. A control can be fixed or patterned:
([Tidal Cycles][3])

```haskell
gain 0.8
gain "1 0.7 0.4"
pan "0 1 0.25 0.75"
```

So this line:

```haskell
d1 $ s "bd sn" # gain "1 0.7" # pan "0 1"
```

builds one final pattern with sound, gain, and pan controls.

## Reading a Full Line

```haskell
d1 $ every 4 rev $ s "bd sn [hh hh]" # gain "1 0.8 0.6"
```

Read it inside out:

1. `"bd sn [hh hh]"` becomes a mini-notation pattern.
2. `s` turns it into a sound control pattern.
3. `gain "1 0.8 0.6"` creates a gain control pattern.
4. `#` combines sound and gain into one pattern.
5. `every 4 rev` reverses that whole pattern every fourth cycle.
6. `$` passes the result to `d1`.

The practical grouping is:

```haskell
d1 (every 4 rev (s "bd sn [hh hh]" # gain "1 0.8 0.6"))
```

not:

```haskell
(d1 $ every 4 rev $ s "bd sn [hh hh]") # gain "1 0.8 0.6"
```

`#` belongs inside the pattern expression. `$` passes completed expressions
into functions such as `d1`, `every`, `slow`, and `fast`.

## Teaching Line

`$` controls how code is grouped. `#` controls how pattern parameters are
combined. In a normal Tidal line, everything to the right of `$` builds one
final pattern, and `#` is used inside that expression to add controls.
