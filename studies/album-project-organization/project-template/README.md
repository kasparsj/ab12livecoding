# Project Template

Copy this folder when starting a final project or small album. Rename the copied
folder to your project title, then update this README before recording.

## Start Here

1. Open `tracks/00-boot.tidal` and evaluate it first.
2. Open `tracks/01-title.tidal` and replace the placeholder title.
3. Add each sample, loop, or stem to `sample-manifest.md` as soon as it enters
   the project.
4. Write performance notes in `performance-cues.md` before rehearsal.
5. Keep production decisions in `production-notes.md`.

Use only stock samples, your own recordings, or material with clear permission.
Do not rely on samples from another artist's project unless the license and
attribution requirements are understood.

## File Map

| File | Purpose |

| [`performance-cues.md`](performance-cues.md) | Rehearsal, load order, section cues, and emergency recovery. |

## Release Boundary

Before sharing a recording or code archive, confirm that every sound source is
listed in `sample-manifest.md`, every attribution is written, and no unknown
download remains in the project.
