## Best sources on $ vs #

### 1. Tidal official tutorial ‚Äî best explanation of `#`

The official Tidal tutorial says `#` joins control patterns, for example:

```haskell
d1 $ sound "bd*4" # crush "4" # speed "2"
```

It explains that this combines the `sound`, `crush`, and `speed` control patterns into one pattern sent to `d1`. ([Tidal Cycles][1])

More importantly, the same tutorial says:

```haskell
d1 $ sound "bd sn drum arpy" # pan "0 1 0.25 0.75"
```

means you are combining a **sound pattern** and a **pan pattern**, and that `#` is shorthand for `|>`. It also says the **left-most pattern determines the rhythmic structure** of the result. ([Tidal Cycles][1])

This is probably the best source for explaining `#`.

---

### 2. Tidal official ‚ÄúPattern Structure‚Äù page ‚Äî best technical explanation of `#`

This is the most precise source. It explains how Tidal combines patterns over time and how structure can come from the left, right, or both sides. It explicitly says:

> `#` is an alias for `|>`.

And `|>` means: take **structure from the left**, but values/controls from the right. ([Tidal Cycles][2])

So this:

```haskell
d1 $ sound "bd sn" # gain "1 0.5 0.8"
```

is roughly:

> Use the event structure of `sound "bd sn"`, then attach/match the `gain` values onto those events.

If you reverse the order:

```haskell
d1 $ gain "1 0.5 0.8" # sound "bd sn"
```

then the **gain pattern** is now on the left, so its structure becomes dominant.

---

### 3. Tidal official ‚ÄúControls‚Äù page ‚Äî what `sound`, `gain`, `pan`, etc. are

This page explains that `gain`, `pan`, `begin`, `end`, `delay`, `shape`, etc. are **control functions**. They turn strings or numbers into **control patterns** that govern how SuperDirt plays sounds. It also states the important principle: **every parameter is patternable**. ([Tidal Cycles][3])

So:

```haskell
gain "1 0.7 0.4"
```

is not just a static effect setting. It is itself a **pattern of gain controls**.

---

### 4. Tidal Course I ‚Äî best explanation of `$` in actual Tidal code

This source directly explains why Tidal code often looks like:

```haskell
d1 $ every 3 (fast 2) $ sound "bd sd ~ cp"
```

It says Tidal functions generally take the pattern they manipulate as their **final input**, which makes chaining easy. It also explains that `$` has very low priority, so everything on its right is worked out first before being passed to the function on the left. ([Tidal Cycles][4])

This is the best source for explaining the shape of Tidal code.

---

### 5. Haskell function application docs ‚Äî what `$` means outside Tidal

Tidal is embedded in Haskell, so `$` comes from Haskell. A Haskell teaching source gives the definition:

```haskell
($) :: (a -> b) -> a -> b
f $ x = f x
```

So `f $ x` just means `f x`, but with low precedence, helping avoid parentheses. ([web.cs.dal.ca][5])

A more beginner-friendly Haskell explanation says `$` takes a function and an input, applies the function, and is useful because its low precedence acts like a grouping separator. ([Monday Morning Haskell][6])

---

### 6. Tidal ‚ÄúWhat is a pattern?‚Äù ‚Äî deeper model

This source explains that under the hood, a Tidal pattern is a **function from time to events**. A pattern like:

```haskell
"1 2 3"
```

is parsed into events across one cycle:

```text
0   ‚Üí 1/3 : "1"
1/3 ‚Üí 2/3 : "2"
2/3 ‚Üí 1   : "3"
```

This is useful for explaining why `#` is not just ‚Äúadd an effect‚Äù; it is combining time-based event patterns. ([Tidal Cycles][7])

## How I would explain `$` vs `#`

### `$` means: ‚Äúsend the whole expression on the right into the function on the left‚Äù

```haskell
d1 $ sound "bd sn"
```

means:

```haskell
d1 (sound "bd sn")
```

So `$` is mostly a parenthesis-saving operator.

This:

```haskell
d1 $ every 3 rev $ sound "bd sn"
```

means:

```haskell
d1 (every 3 rev (sound "bd sn"))
```

So `$` organizes **function application**.

---

### `#` means: ‚Äúcombine this control pattern with another control pattern‚Äù

```haskell
sound "bd sn" # gain "1 0.7"
```

means:

> make a sound pattern, make a gain pattern, and combine them into one control pattern.

So `#` organizes **pattern parameters**.

---

## The full structure of a normal Tidal line

Take this:

```haskell
d1 $ every 4 rev $ sound "bd sn [hh hh]" # gain "1 0.8 0.6"
```

You can read it as:

```text
d1 $
  every 4 rev $
    sound "bd sn [hh hh]"
    # gain "1 0.8 0.6"
```

Meaning:

1. `"bd sn [hh hh]"` becomes an inner mini-notation pattern.
2. `sound ...` turns it into a **sound control pattern**.
3. `gain "1 0.8 0.6"` becomes a **gain control pattern**.
4. `#` combines those control patterns into one larger control pattern.
5. `every 4 rev` transforms that whole pattern every 4 cycles.
6. `$` passes the result into `d1`.
7. `d1` plays/sends that final pattern to the first stream/orbit.

The most important practical rule:

```haskell
d1 $ sound "bd sn" # gain "1 0.5 0.8"
```

usually reads as:

```haskell
d1 (sound "bd sn" # gain "1 0.5 0.8")
```

not:

```haskell
(d1 $ sound "bd sn") # gain "1 0.5 0.8"
```

So **`#` belongs inside the pattern**, while **`$` passes the completed pattern into a function like `d1`, `every`, `slow`, `fast`, etc.**

## Short explanation

> `$` is Haskell‚Äôs ‚Äúapply this whole thing to the left‚Äù operator. It helps avoid parentheses.
> `#` is Tidal‚Äôs ‚Äúcombine control patterns‚Äù operator. It attaches parameters like `gain`, `pan`, `speed`, `n`, `begin`, and `end` to a sound/note pattern.
> In a normal Tidal line, the expression to the right of `$` builds one final pattern, and `#` is used inside that expression to combine the pattern‚Äôs controls.
